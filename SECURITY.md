# Security Policy

This project includes authentication, an admin dashboard, and a backend API.
Security issues should be handled responsibly and privately.

---

## Reporting a Vulnerability

Please **do not open public GitHub issues** for security-related problems.

Instead, report vulnerabilities privately using one of the following methods:
- GitHub Security Advisories (preferred, if enabled for this repository)
- Directly contacting the maintainer with detailed information

---

## What to Include

When reporting a vulnerability, please include:
- A clear description of the issue
- Affected components (frontend, backend, admin, auth, etc.)
- Steps to reproduce the issue
- Potential impact or severity
- Suggested fix or mitigation (if known)

---

## Scope

This security policy applies to:
- Authentication and authorization logic
- Admin dashboard functionality
- API endpoints and data handling
- Configuration or deployment-related security issues

---

## Disclosure

Please allow reasonable time for the issue to be reviewed and addressed
before sharing details publicly.

The maintainer reserves the right to determine the severity and handling
of reported security issues.
